# Анализ Cybos vs наш бот

Репозиторий: https://github.com/Gerstep/cybos
Дата анализа: 2026-03-06

## Стоит внедрить (высокий приоритет)

### 1. Leverage Scoring — приоритизация задач по весам
У них каждая задача получает числовой "leverage score" по YAML-правилам: дедлайн близко → +30, блокирует других → +25, quick win (<15 мин) → +20 и т.д. Утренний брифинг сортирует по этому скору вместо плоского списка.

**Наш бот**: сейчас Claude просто смотрит на список и сам решает приоритеты. С leverage scoring утренний брифинг станет объективнее и стабильнее.

### 2. Entity Resolution — умный поиск контактов
Cybos использует SQLite FTS5 для fuzzy-поиска людей: "Шмаков", "шмаков", "Антон Шмаков" — всё найдёт одного человека. Нормализация имён, алиасы.

**Наш бот**: CRM ищет по точному вхождению строки. Claude компенсирует, но нативный fuzzy search будет надёжнее и быстрее.

### 3. Activity Logging — лог всех действий
Каждое действие бота логируется в structured формате: что сделал, когда, результат. Потом используется для self-improvement и аналитики ("за неделю создал 15 задач, закрыл 8").

**Наш бот**: логов действий нет. Нельзя посмотреть статистику или понять паттерны.

## Интересно, но не срочно

### 4. Parallel Agent Execution
У них 13 специализированных агентов (research, messaging, content) которые могут работать параллельно. Один агент ресёрчит, другой пишет письмо.

Для нашего бота overkill пока — у нас один пользователь, задачи sequential.

### 5. Workspace-per-Session
Каждый ресёрч/встреча создаёт отдельную "workspace" папку со всеми артефактами. Удобно для изоляции.

У нас уже есть `meetings/` и `kb/project/` — похожая структура, но менее формализованная.

### 6. Quality Review Loop
После важного анализа запускается "review agent" который проверяет качество и просит доработать (до 3 итераций).

Полезно для PRD/документов, но добавляет latency и стоимость API.

### 7. MCP Integrations (8 штук)
Exa и Perplexity для web research, Gmail для email, Telegram для сообщений, Google Calendar — всё через MCP protocol.

Мы можем добавить web search для ресёрча (Perplexity API) и Google Calendar для дедлайнов. Но это отдельный проект.

## Что у нас уже лучше

- **Smart routing** — у них жёсткие slash-команды (23 штуки), у нас свободный ввод с AI-роутингом. Удобнее для пользователя.
- **Evening review** — у них нет аналога, только morning brief.
- **Встречи → задачи** — у них встречи не парсятся автоматически в задачи/CRM.

## Рекомендация

Начать с **leverage scoring** — максимальный эффект при минимальных усилиях. Добавляем YAML-конфиг с правилами приоритизации, и утренний брифинг сразу станет умнее.
